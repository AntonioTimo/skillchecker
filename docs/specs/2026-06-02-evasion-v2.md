# Spec — Evasion v2: normalization + homoglyph domains (Phase E)

- **Date:** 2026-06-02
- **Status:** Proposed (awaiting review)
- **Target version:** 1.5.0
- **Branch:** `feat/evasion-v2`
- **Phase:** E — the first v3 increment.

---

## 1. Problem

Three evasion gaps survive v2:

- **Compatibility / fullwidth Unicode.** `ｃｕｒｌ … | sh` (fullwidth latin) or a
  math-styled `𝖼𝗎𝗋𝗅` render like ASCII but the regex (which matches ASCII
  `curl`) misses them. The Unicode pass (B) flags bidi / zero-width / homoglyph
  but does **not** NFKC-normalize, so compatibility-char commands slip through.
- **IDN / punycode domains.** `https://xn--80ak6aa92e.com/` (punycode) passes the
  exfil rules; a homoglyph host reads as a trusted brand.
- **Cloud-metadata SSRF.** `http://169.254.169.254/latest/meta-data/…` (IAM
  credential theft) is skipped by `HI019`'s link-local guard — a gap found after
  v1.4.0.

## 2. Approach

- Scan an **NFKC-normalized** copy of each code / inline-code target in addition
  to the raw text, so fullwidth / compatibility dangerous patterns surface.
- Add `CR038` (cloud-metadata SSRF) and `HI022` (punycode / IDN host). Verify a
  raw homoglyph host is already caught by `UNI004`; add a test either way.

## 3. Design

- **`scan_file`:** for each scannable target, also test the rules against
  `unicodedata.normalize("NFKC", target)`. A match found only on the normalized
  form is reported with a "(via NFKC — text uses fullwidth/compatibility
  characters)" note. No double-report when `normalized == raw`. Normalization is
  **escalate-only** — it can reveal a dangerous pattern, never suppress one.
- **`CR038`** — `169\.254\.169\.254` / `metadata\.google\.internal` /
  `100\.100\.100\.200` (Alibaba) → CRITICAL. A dedicated rule, so `HI019`'s
  link-local skip does not hide the metadata endpoint.
- **`HI022`** — `https?://(?:[^/\s]*\.)?xn--…` → HIGH (IDN / punycode host;
  homoglyph phishing / C2 channel).

## 4. False-positive management

- NFKC re-scan only **escalates** — legit compatibility text (`½`, `™`, the `ﬁ`
  ligature, CJK) does not match a dangerous regex after normalization.
- `HI022` is HIGH (review), since a legitimate IDN is rare in a skill's network
  call.
- The existing clean fixtures and the self-audit must stay green (no compat chars
  or punycode there); NFKC of Cyrillic/em-dash is a no-op, so the Russian docs and
  the scanner's own source are unaffected.

## 5. Documentation & tests

- `references/red-flags.md` + `THREAT_MODEL.md` rows; `CHANGELOG.md` `[1.5.0]`;
  a note in `SKILL.md` Step 6.7.
- **RED:** `examples/evil-evasion/` — fullwidth `ｃｕｒｌ｜sh`, a math-styled `exec`,

- **REFACTOR:** `examples/clean-evasion/` — legit compatibility text (`½`, `™`, a

- **CI:** `evil-evasion` must exit 3 with `CR038`+`HI022`; `clean-evasion` exit 0.
