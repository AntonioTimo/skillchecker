## 1. Problem

The static layer is line-by-line regex. An author who has read the rules dodges
them with trivial surface changes:

- **Alias a builtin:** `e = eval; e(payload)` â€” `HI006` (`\beval\s*\(`) never fires.
- **Split a name:** `getattr(os, "sys" + "tem")("id")` â€” no rule resolves `os.system`.
- **Multi-line call:** `subprocess.run(\n    cmd,\n    shell=True,\n)` â€” `HI005`/`CR018`
  are applied per line and never see the call and `shell=True` on the same line.
- **Build a string from chars:** `exec("".join(chr(c) for c in payload))`.

`THREAT_MODEL.md` Â§ "What is out of scope" #4 names this as the headline
limitation and points at *"AST-based analysis (planned for v2.0.0)"*. This phase
is that analyzer.

## 2. Why AST

`ast.parse(source)` builds a syntax tree **without executing the code** (unlike
`import`/`eval`). The tree is immune to surface obfuscation: whitespace, line
breaks, and string-vs-call layout are irrelevant â€” `subprocess.run(...)` is a
single `Call` node whether it spans one line or ten. Walking it with
`ast.NodeVisitor` lets us resolve call targets and argument shapes structurally.

A bonus: AST distinguishes a **string literal** `"eval("` from an actual `eval(...)`
**call**, so it does not reproduce the regex pass's self-audit false positives on
this very scanner (whose rule strings contain `eval`, `exec`, etc. as data).

## 3. Goals / Non-goals

**Goals**
- A new `ast_scan()` pass for `.py` files, **additive** to the regex pass.
- Catch the evasions above: dynamic `eval`/`exec`/`compile`, aliased builtins,
  `subprocess(..., shell=True)` at any line layout, `pickle`/`marshal`/`yaml.load`,
  dynamic `getattr`/`__import__`, and exec/eval fed by constructed strings.
- **Degrade gracefully:** if `ast.parse` fails (syntax error, Python 2, non-Python
  content), the file still gets the regex pass; AST simply adds nothing.
- **Never execute code.**

**Non-goals (YAGNI for Phase A)**
- No full data-flow / taint analysis. Alias tracking is a single bounded pass over
  direct `name = builtin` assignments â€” not interprocedural.
- No JS/TS AST (regex still covers JS; that is Phase D territory at most).
- No replacement of the regex pass â€” AST complements it.

## 4. Design

New section in `scripts/scan.py`: `ast_scan(path, rel) -> list[Finding]`, called
from `main()` for every `.py` file (alongside `scan_file`). A `NodeVisitor`
subclass walks the tree. New rule-ID namespace `AST0xx`, each carrying a severity
so counts/verdict are unchanged.

| Rule | What it catches | Severity |
|---|---|---|
| `AST001` | `eval` / `exec` / `compile` call with a **non-literal** argument | CRITICAL |
| `AST002` | call to an **alias** of eval/exec/compile (`e = exec; e(x)`) | CRITICAL |
| `AST003` | `os.system` / `os.popen` / `subprocess.*` with `shell=True` (any layout) | CRITICAL if command non-literal, else HIGH |
| `AST004` | `pickle.loads` / `marshal.loads` | CRITICAL |
| `AST005` | `yaml.load` without `SafeLoader` / not `yaml.safe_load` | HIGH |
| `AST006` | `getattr(<obj>, <non-literal>)` â€” dynamic attribute dispatch | HIGH |
| `AST007` | `__import__(<non-literal>)` / `importlib.import_module(<non-literal>)` | HIGH |
| `AST008` | exec/eval fed by `chr()` / `bytes.fromhex` / `codecs.decode` / `"".join(...)` | CRITICAL |

**Resolution helpers**
- `_call_dotted_name(call)` â†’ the dotted name of a `Call`'s func (`os.system`,
  `subprocess.run`, `eval`) by walking `Attribute`/`Name` chains.
- **alias map:** a pre-pass over `Assign` nodes whose `value` is a bare `Name`
  referencing a known-dangerous builtin (`eval`/`exec`/`compile`); record
  `target_id â†’ builtin`. `AST002` flags later calls whose func is a `Name` in the
  map.
- **"non-literal" argument:** the argument node is not an `ast.Constant` (nor a
  literal container of constants). `eval("1+1")` (constant) is left to the regex
  pass; `AST001` targets the dynamic case regex cannot see.

**False-positive guards**
- `AST003` with a constant string command (`subprocess.run("ls -l", shell=True)`)
  â†’ HIGH (bad practice) not CRITICAL.
- `subprocess.run(["ls", "-l"])` (list args, no `shell=True`) â†’ **no finding**
  (safe by construction).
- `getattr(obj, "literal_attr")` (constant attribute) â†’ **no finding**.
- `json.loads(...)`, `yaml.safe_load(...)` â†’ **no finding**.

Findings use the node's real `.lineno`.
